# 升降级规则

## 等级

- `L0 raw`：原始材料，只是证据，不算正式记忆。
- `L1 candidate`：候选记忆，尚未权威化。
- `L2 accepted`：已接受，进入项目记忆或结构化记忆。
- `L3 promoted`：长期稳定、跨任务可复用，可进入核心记忆。
- `L4 active`：高优先级默认规则，进入 `10_核心记忆/ACTIVE.md`。

## L0 -> L1

满足任一条件即可进入候选：

- 可能反复影响未来工作。
- 纠正了之前的错误。
- 定义了项目目标、指标、SQL 口径或重要限制。
- 暴露了可复发的数据、工具、流程问题。
- 用户明确说“记下来”。

## L1 -> L2

候选可以接受的条件：

- 不是一次性临时想法。
- 来源清楚。
- 适用范围清楚。
- 未来同类任务会用到。
- 没有和现有权威记忆冲突。
- 不包含敏感原文或凭证。

## L2 -> L3

只有满足以下条件才提升到核心记忆：

- 跨多个任务或项目复用。
- 已经被后续任务验证过。
- 不是局部项目细节。
- 未来默认读取它会提升结果质量。
- 写入后不会让核心记忆变臃肿。

## L3 -> L4

只有“默认必须遵守”的规则才进 `ACTIVE.md`：

- 违反后会明显影响质量、安全或判断。
- 适用于绝大多数未来任务。
- 用户明确认可。
- 不是项目局部规则。
- 不是临时偏好。

## 降级

- `ACTIVE -> LEARNINGS`：规则不再需要默认强制执行，但仍有参考价值。
- `CORE -> PROJECT`：其实只是某个项目的事实，不应全局化。
- `accepted -> archive`：已过期、被新事实替代、长期不用。
- `candidate -> rejected`：无法验证、重复、过窄、含敏感信息，或只是当时想法。
