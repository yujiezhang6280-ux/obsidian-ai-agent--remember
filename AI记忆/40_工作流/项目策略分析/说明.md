# 项目策略分析

这个方法层用于适配项目记忆 Skill 的通用语义。

## 定位

- 项目记忆 Skill 是可选增强层，不是本模板的依赖。
- 中文 `_项目记忆/` 是项目事实、指标和 SQL 逻辑的真相源。
- 本文件只说明方法，不保存项目事实，不覆盖 `30_项目记忆/.../_项目记忆/`。

## 兼容规则

外部工具或旧说明提到英文项目记忆时，按中文正式结构读取：

- `_project_memory/` -> `_项目记忆/`
- `00-critical.md` -> `00-关键上下文.md`
- `parents.md` -> `父项目.md`
- `01-background.md` -> `01-背景.md`
- `02-data-dictionary.md` -> `02-数据字典.md`
- `03-metric-logic.md` -> `03-指标口径.md`
- `04-sql-notes.md` -> `04-SQL笔记.md`
- `97-retrospective.md` -> `97-复盘.md`
- `98-chat-intake.md` -> `98-对话沉淀.md`
- `99-index.md` -> `99-索引.md`

## 读取顺序

1. 当前用户消息。
2. 当前项目 `_项目记忆/00-关键上下文.md`。
3. 当前项目 `父项目.md`，如存在且相关。
4. 当前项目 `99-索引.md`。
5. 只读取当前任务需要的项目细节文件。
6. 父项目只在涉及共享背景、共享指标或跨项目决策时读取。

## 规则

- 项目事实写入项目 `_项目记忆/`。
- `父项目.md` 只记录继承关系，不复制父项目事实。
- 方法文件只指导推理，不保存项目事实。
- SQL 问题先翻译业务含义，再下结论。
- 旧 `_project_memory/` 只做兼容读取，不作为新项目默认结构。
