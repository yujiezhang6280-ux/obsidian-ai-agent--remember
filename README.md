# Obsidian AI Agent Memory Template

一套给 Codex、Claude、Gemini 和其他 AI Agent 共用的 Obsidian 记忆库模板。

核心原则：

- 文件夹是长期真相源，数据库、Dataview、Bases 或表格只作为索引和视图。
- Agent 默认先写候选，不直接污染权威记忆。
- 项目事实留在项目记忆里，只有跨任务稳定复用的规则才提升到核心记忆。
- 来源、日常记录、附件和隐私边界默认不作为权威记忆。

## 适合谁

- 想让多个 AI Agent 共用同一套长期记忆的人。
- 想把项目上下文、指标口径、SQL 逻辑、复盘经验沉淀到 Obsidian 的团队。
- 需要一套可审核、可迁移、可降级的记忆治理规则的人。

## 快速开始

1. 下载或 fork 本仓库。
2. 把整个目录复制到你的 Obsidian vault 中。
3. 打开 `说明.md`，把 `<你的 Obsidian vault 路径>` 替换为本机真实路径。
4. 按需把 `AGENTS.md`、`CLAUDE.md`、`GEMINI.md` 复制或链接到对应 Agent 会读取的位置。
5. 新项目记忆从 `AI记忆/30_项目记忆/_模板/项目记忆模板/` 复制一份到 `AI记忆/30_项目记忆/进行中/<项目名>/`。

## 默认目录

```text
AGENTS.md      Codex / 通用 Agent 入口
CLAUDE.md      Claude 入口
GEMINI.md      Gemini 入口
说明.md        整套记忆库总说明
AI记忆/        正式记忆内容根目录
docs/          安装、发布和安全说明
```

`AI记忆/` 下的主要职责：

```text
00_控制规则/      记忆政策、写入规则、升降级规则、隐私规则
10_核心记忆/      全局长期规则、用户画像、稳定偏好、重要纠错
20_结构化记忆/    已审核的原子事实、事件、决策、洞察、视图和候选
30_项目记忆/      具体项目上下文，使用 _项目记忆 结构
50_来源归档/      来源材料和迁移备份，默认不是权威记忆
60_日常记录/      日记、复盘、周/月总结，默认不自动提升
70_知识库/        概念、人物、公司、产品、工具等通用知识
80_附件/          图片、PDF、导出文件，不默认批量读取
90_隐私边界/      禁止读取、禁止索引、禁止长期保存的边界
```

## 不要提交的内容

本仓库是模板，不应该包含你的真实记忆内容。默认 `.gitignore` 已忽略：

- `AI记忆/50_来源归档/` 中的来源材料
- `AI记忆/60_日常记录/` 中的日记和复盘原文
- `AI记忆/80_附件/` 中的附件
- inbox、待确认和归档候选
- 常见密钥、凭证、环境文件

团队使用时，建议把真实记忆库保留在私有仓库或本地同步盘里；公开仓库只维护这套模板和规则。
