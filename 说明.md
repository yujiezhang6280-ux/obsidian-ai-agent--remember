# Agent 记忆库

这是一套给 Codex、Claude、Gemini 和后续 AI Agent 共用的长期记忆系统。

核心原则：**文件夹是长期真相源，数据库、Dataview、Bases 或表格只做索引和视图。**

## 一句话逻辑

```text
入口文件负责告诉 Agent 怎么读写；
控制规则负责判断什么能成为记忆；
核心记忆保存长期稳定规则；
结构化记忆保存已审核的事实、事件、决策、洞察；
项目记忆保存具体项目上下文；
工作流只保存方法，不覆盖项目事实；
来源和日常记录只做证据，不默认当作权威记忆。
```

## 安装路径

把 `<你的 Obsidian vault 路径>` 替换成你的真实路径。

```text
<你的 Obsidian vault 路径>/Agent记忆库
```

## 什么是可用记忆

默认可用：

```text
10_核心记忆/
20_结构化记忆/事实/
20_结构化记忆/事件/
20_结构化记忆/决策/
20_结构化记忆/洞察/
20_结构化记忆/_视图/
30_项目记忆/.../_项目记忆/
```

方法层按需可读：

```text
40_工作流/
```

`40_工作流/` 只保存分析方法、检查流程和输出习惯，不保存项目事实，不覆盖 `30_项目记忆/.../_项目记忆/`。

默认不可直接当真：

```text
20_结构化记忆/_收件箱/
20_结构化记忆/_待确认/
20_结构化记忆/_归档/
50_来源归档/
60_日常记录/
80_附件/
90_隐私边界/
未审核导入、聊天记录、日志、原始导出、临时草稿
```

判断标准：

```text
可用 = 已审核、稳定、范围明确、未来会影响判断
不可用 = 原始、临时、未审核、敏感、过期、只作为证据
```

## 默认读取顺序

Agent 应该按最小必要原则读取：

```text
1. 读取对应入口文件：AGENTS.md / CLAUDE.md / GEMINI.md
2. 读取 AI记忆/00_控制规则/记忆策略.md
3. 读取 AI记忆/00_控制规则/写入策略.md
4. 读取 AI记忆/10_核心记忆/ACTIVE.md
5. 读取 AI记忆/10_核心记忆/PROFILE.md
6. 项目任务再读取对应项目 _项目记忆/00-关键上下文.md 和 99-索引.md
7. 只按当前任务读取目标文件
```

不要默认全库扫描。

## 项目记忆兼容

中文 `_项目记忆/` 是正式结构。外部项目记忆 Skill 或旧项目说明提到 `_project_memory/`、`00-critical.md`、`parents.md`、`99-index.md` 等英文名时，按 `AI记忆/30_项目记忆/说明.md` 的映射读取。

本模板不依赖任何私有 Skill；Skill 只是可选增强层。
