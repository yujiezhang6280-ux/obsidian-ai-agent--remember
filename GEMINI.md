# Gemini 记忆入口

主记忆库路径：

```text
<你的 Obsidian vault 路径>/Agent记忆库
```

使用和其他 AI Agent 相同的记忆规则：

- 先读取 `说明.md`。
- 读取 `AI记忆/00_控制规则/记忆策略.md`。
- 读取 `AI记忆/00_控制规则/写入策略.md`。
- 读取 `AI记忆/10_核心记忆/ACTIVE.md` 和 `AI记忆/10_核心记忆/PROFILE.md`。
- 项目任务使用 `AI记忆/30_项目记忆/.../_项目记忆/`。
- 项目路径支持 `AI记忆/30_项目记忆/<状态>/[<项目组>/]<项目名>/_项目记忆/`；没有 `_项目记忆/` 的中间目录是项目组容器，应继续向下一层查找。

## 项目记忆兼容规则

Gemini 不依赖 Codex 或其他 Agent 的私有 Skill，但应遵守同一套项目记忆语义。

中文文件名是正式结构。外部说明或旧工具提到英文名时，按下面映射读取：

- `_project_memory/` = `_项目记忆/`
- `00-critical.md` = `00-关键上下文.md`
- `parents.md` = `父项目.md`
- `01-background.md` = `01-背景.md`
- `02-data-dictionary.md` = `02-数据字典.md`
- `03-metric-logic.md` = `03-指标口径.md`
- `04-sql-notes.md` = `04-SQL笔记.md`
- `97-retrospective.md` = `97-复盘.md`
- `98-chat-intake.md` = `98-对话沉淀.md`
- `99-index.md` = `99-索引.md`

`AI记忆/40_工作流/` 是方法层，不保存项目事实，不覆盖 `30_项目记忆/.../_项目记忆/`。

不要默认把来源、日常记录、候选、待确认判断、隐私文件或归档当作权威记忆。
