# Agent 记忆入口

这个库是 Codex 和其他 AI Agent 共用的长期记忆库。

## 主路径

```text
<你的 Obsidian vault 路径>/Agent记忆库
```

## 读取顺序

1. 读取 `说明.md`，理解整套记忆的核心逻辑和升降级策略。
2. 读取 `AI记忆/00_控制规则/记忆策略.md`。
3. 读取 `AI记忆/00_控制规则/写入策略.md`。
4. 读取 `AI记忆/10_核心记忆/ACTIVE.md`。
5. 读取 `AI记忆/10_核心记忆/PROFILE.md`。
6. 如果任务涉及具体项目，读取相关项目的 `AI记忆/30_项目记忆/.../_项目记忆/00-关键上下文.md`，必要时再读 `父项目.md` 和 `99-索引.md`。
7. 只读取当前任务需要的最少文件。

## 项目记忆 Skill 兼容

本模板不依赖任何私有 Skill。若当前 Agent 可用项目记忆 Skill，应让该 Skill 按本模板的中文 `_项目记忆/` 结构读写。

旧 `_project_memory/` 只做兼容读取来源，不作为新项目默认结构。两者并存时，以中文 `_项目记忆/` 为权威，并提示重要冲突。

## 写入规则

除非用户明确要求，不要直接覆盖核心记忆。

默认写入位置：

- 全局记忆候选：`AI记忆/20_结构化记忆/_收件箱/codex/操作/`
- 项目记忆：仅当用户要求更新项目记忆，或当前任务明确产生长期项目上下文时，写入对应项目 `_项目记忆/`。

## 安全规则

- 不读取或摘要路径包含 `password`、`credentials`、`secrets`、`token`、`账号密码` 等敏感名称的文件。
- 不把原始日记全文复制进记忆。
- `AI记忆/50_来源归档/`、`AI记忆/60_日常记录/`、`_收件箱/`、`_待确认/`、`AI记忆/90_隐私边界/` 默认不是权威记忆，除非用户明确要求检查。
- `AI记忆/40_工作流/` 是方法层，只指导流程，不保存项目事实，不覆盖 `30_项目记忆/.../_项目记忆/`。
